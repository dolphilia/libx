# تحرير وتنسيق المحتوى

بعد إنشاء الصفحات، نحتاج إلى التأكد من أن التحرير يتم بصورة متسقة مع بقية اللغات وأن عناصر RTL تعمل حتى داخل الأمثلة المعقدة.

## أقسام مقترحة للصفحة

1. **مقدمة قصيرة** تشرح الهدف من التعديل الجديد.
2. **كتل تعليمية** تضم خطوات مرقمة أو قوائم تحقق.
3. **أمثلة عملية** توضح الفارق بين الاتجاهين LTR/RTL.

## التعامل مع الأكواد

تحافظ الطبقة الجديدة من CSS على اتجاه يسار-إلى-يمين داخل كتل الأكواد حتى لو كانت الصفحة باللغة العربية. لذا يمكن إضافة مثال:

```ts
import { initializeConfig } from '@docs/project-config';

await initializeConfig();
console.log('تم تحميل الإعدادات بنجاح');
```

لاحظ أن النص المحيط سيبقى RTL بينما تبقى الأكواد بمحاذاة يسار لسهولة القراءة.

## تحديث الروابط الداخلية

- استخدم المسار الأساسي `projectConfig.baseUrl` عند إنشاء روابط داخلية.
- تأكد من أن الحقول `prev` و `next` (إن وُجدت في frontmatter) تشير إلى ملفات موجودة فعلًا كي لا تفشل عملية الدمج مع الصفحات الآلية.

## خطوات التحقق

1. شغّل `pnpm --filter=sample-docs dev` ثم افتح `/docs/sample-docs/v2/ar/01-guide/03-editing-documents`.
2. تأكد من أن شريط التقدم والصفحات السابقة/اللاحقة تظهر بالعربية وتعكس اتجاه RTL.
3. أعد بناء الموقع: `pnpm --filter=apps-sample-docs build` للتأكد من أن المسارات الجديدة تدخل في التصدير النهائي.

باتباع هذه الخطوات ستصبح لديك سلسلة صفحات عربية متتالية يمكن استخدامها لاختبار سلوك التصفّح والصفحات المجاورة.
